# 一、项目概述

## 1\.1 项目背景

现代人群对健康饮食、食材营养、科学烹饪的需求持续提升，但缺少一体化线上饮食管理平台。普通用户无法便捷查询食材营养、查看健康食谱、阅读食品科普资讯；管理员难以统一维护食材分区、食材库存、食谱、资讯、轮播图等数据。
为解决上述痛点，开发**健康饮食管理系统**，采用前后端分离架构，划分**用户前台浏览端**与**管理员后台管理端**，实现食材、食谱、资讯、食材烹饪记录全生命周期数字化管理，为大众提供健康饮食查询工具，同时支持管理员后台统一数据维护。

## 1\.2 项目目标

1. 采用 SpringBoot 后端 \+ Vue 前端标准前后端分离架构，接口统一使用 RESTful 规范；

2. 区分两类角色：普通用户、系统管理员，实现权限隔离；

3. 用户前台功能：首页轮播展示、食材百科检索、食谱浏览、食品资讯阅读、个人中心（食材新增 / 修改 / 删除、食材烹饪记录管理）、账号注册登录；

4. 管理员后台功能：食品资讯管理、食材百科管理、轮播图管理、食材位置分区管理、食谱信息管理、系统用户管理、食材烹饪记录管理；

5. 支持图片上传（食材封面、资讯配图、首页轮播图、用户头像）、模糊搜索、分页列表、新增 / 编辑 / 删除 / 详情查看；

6. 使用 JWT 实现登录鉴权，全局异常捕获、统一接口返回格式，保障系统数据安全与稳定性。

## 1\.3 用户角色说明

1. **普通用户**：前台页面注册登录，可浏览食材、食谱、食品资讯；可自主录入自有食材、管理食材烹饪记录，仅能操作本人数据，无后台数据管理权限。

2. **系统管理员**：后台登录，拥有全量数据管理权限，维护所有食材、食谱、资讯、分区、轮播图、用户账号数据。

# 二、系统需求分析

## 2\.1 功能性需求

### （1）登录注册模块

- 用户注册：填写账号、密码、确认密码、姓名、上传头像、选择性别、填写手机号，账号唯一校验；

- 系统登录：区分管理员 / 普通用户身份登录，校验账号密码；

- 退出登录：清除本地 Token，跳转登录页面；

- 个人中心：用户可管理自己录入的食材、烹饪记录。

### （2）用户前台功能

1. 前台首页：顶部全屏饮食主题轮播图，下方食谱推荐轮播卡片，左侧固定导航栏（首页、食材百科、食谱信息、食品资讯、个人中心）；

2. 食材百科页面：顶部食材名称搜索框，网格卡片展示所有食材，点击卡片查看食材详情；

3. 食品资讯页面：浏览管理员发布的全部健康饮食科普资讯；

4. 食谱信息页面：分类展示健康食谱，支持点击查看完整食谱内容；

5. 个人中心页面：

    - 新增食材弹窗：填写食材存放位置、食材数量、保质期、关联用户信息、录入营养价值、上传食材图片；

    - 食材详情页：展示食材图片、存放分区、数量、保质期、营养说明、录入人信息；支持食材烹饪、修改、删除操作；

    - 食材烹饪：生成食材烹饪记录，后台统一管理。

### （3）管理员后台功能

后台左侧菜单栏，包含全部数据管理模块：

1. 新闻资讯管理 \- 食品资讯：资讯标题模糊搜索，分页列表，支持新增、修改、删除资讯；

2. 食材百科管理：全量食材数据维护，搜索、新增、编辑、删除；

3. 轮播图管理：维护前台首页轮播图片，上传图片、设置轮播名称、排序；

4. 食材位置管理：食材存放分区（A 区 / B 区 / C 区等）管理，新增、修改、删除分区；

5. 食谱信息管理：所有健康食谱统一维护；

6. 系统用户管理：管理前台注册普通用户账号；

7. 食材烹饪管理：查看所有用户提交的食材烹饪记录，支持按食材名称搜索、批量勾选、详情 / 新增 / 修改 / 删除操作；

## 2\.2 非功能性需求

1. **性能需求**：所有列表分页加载，单页数据≤10 条，接口响应时间≤500ms；图片上传做大小压缩限制；

2. **安全需求**：JWT 无状态登录鉴权，未登录拦截所有业务接口；密码 MD5 加密存储；后台接口禁止普通用户访问；

3. **易用性需求**：表单必填项标红校验，弹窗提示操作成功 / 失败，搜索、分页、操作按钮直观；

4. **兼容性需求**：前端页面适配 Chrome、Edge 主流浏览器；

5. **可扩展需求**：模块化分层开发，后续可拓展饮食推荐、热量计算、饮食打卡模块。

7. 接口文档：Knife4j（Swagger）可视化在线接口调试；

4. 状态管理：Pinia，存储登录用户信息、全局 Token；

5. 网络请求：Axios 二次封装，统一请求 / 响应拦截，自动携带 Token、统一错误弹窗；

6. 组件工具：ElUpload 图片上传、ElTable 分页表格、ElDialog 弹窗、ElCarousel 轮播组件；

7. 开发工具：VS Code。

## 3\.3 开发 \& 运维工具

1. 后端开发 IDE：IntelliJ IDEA 2022；

2. 数据库可视化：Navicat Premium；

3. 版本控制：Git \+ Gitee；

4. 服务器部署：CentOS7、JDK1\.8、Nginx 反向代理。

# 四、系统整体架构

系统采用标准**前后端分离架构**，前端仅负责页面渲染与交互，后端负责业务逻辑、数据持久化、接口权限校验，两端通过 HTTP\+JSON 交互，完全解耦。

## 4\.1 前端分层结构

1. **视图层 View**：前台用户页面、管理员后台页面、新增 / 编辑弹窗、详情弹窗；

2. **路由层 Router**：全局路由守卫，校验 Token，未登录跳转登录页，区分前台 / 后台访问权限；

3. **网络请求层 Axios**：统一封装基础请求地址，请求拦截自动添加 Token，响应拦截捕获 401 登录过期、500 服务异常；

4. **状态管理 Pinia**：全局存储登录用户账号、用户类型、Token；

5. **工具层 Utils**：时间格式化、图片上传处理、表单校验、全局提示弹窗。

## 4\.3 完整数据交互流程

1. 用户在 Vue 页面执行操作（登录、查询食材、新增食材、管理烹饪记录）；

2. Axios 请求拦截器自动在请求头携带 Token 令牌；

3. HTTP 请求发送至 SpringBoot 后端对应接口；

4. JWT 全局拦截器校验 Token 有效性，未登录 / 过期直接返回 401；

5. Controller 接收前端参数，调用对应 Service 执行业务逻辑；

6. Service 调用 Mapper 操作 MySQL 数据库，完成增删改查；

7. 后端封装统一 JSON 格式结果返回前端；

8. Axios 响应拦截统一处理成功提示、异常报错，前端渲染页面数据。

## 模块 1：登录注册模块

### 5\.1 系统登录页面

页面功能：输入账号、密码，下拉选择用户类型（管理员 / 普通用户）；表单非空校验；登录成功存储 Token，跳转对应首页（管理员后台 / 用户前台）；提供注册页面跳转入口。

### 5\.2 用户注册页面

表单字段：用户账号、用户密码、确认密码、用户姓名、头像上传、性别下拉选择、手机号码；
业务规则：

1. 用户账号唯一，重复账号注册拦截；

2. 两次输入密码必须一致；

3. 头像图片格式限制（jpg/png），后端接收并保存图片路径；

4. 注册成功自动跳转登录页面。

## 模块 2：用户前台功能模块

### 2\.1 前台首页

页面结构：

1. 顶部全屏轮播图：展示后台上传的饮食宣传轮播图；

2. 左侧侧边固定导航栏：首页、食材百科、食谱信息、食品资讯、个人中心；

3. 中部食谱轮播卡片：横向滚动展示推荐健康食谱；

4. 右上角系统时间、退出登录按钮。

### 2\.2 食材百科列表页面

功能点：

1. 顶部搜索框：输入食材名称模糊检索；

2. 网格卡片布局：每张卡片展示食材封面图 \+ 食材名称；

3. 分页切换；点击食材卡片跳转食材详情页面。

### 2\.3 食材详情页面

展示内容：食材大图、存放分区（A/B/C 区）、食材库存数量、保质期、营养价值描述、录入用户账号与姓名；
操作按钮：食材烹饪（生成烹饪记录）、修改食材信息、删除本人食材数据。

### 2\.4 个人中心 \- 新增食材弹窗

弹窗表单必填项：食材存放位置下拉、食材数量、保质期日期选择；
选填项：用户账号 / 姓名自动填充、营养价值文本框、食材图片上传；
操作按钮：取消、保存提交，提交后食材数据入库。

### 2\.5 食品资讯、食谱信息页面

1. 食品资讯：分页展示管理员发布的健康饮食科普文章；

2. 食谱信息：分类展示全部健康食谱，点击查看完整制作步骤、食材搭配。

## 模块 3：管理员后台功能模块

### 3\.1 后台首页

页面布局：左侧树形菜单栏、顶部右上角管理员头像与退出按钮、主内容欢迎页面。
左侧菜单包含：新闻资讯管理、食材百科管理、轮播图管理、食材位置管理、食谱信息管理、系统用户管理、食材烹饪管理。

### 3\.2 食品资讯管理页面

功能：资讯标题搜索框、分页表格展示资讯列表；支持新增资讯、修改已有资讯、删除资讯；表格展示序号、资讯标题、资讯配图。

### 3\.3 轮播图管理页面

维护前台首页轮播素材，表格展示轮播名称、轮播图片；管理员可新增、修改、删除轮播图，控制前台首页展示内容。

### 3\.4 食材位置管理页面

管理食材存放分区（A 区、B 区、C 区等）；顶部分区名称搜索框，分页列表；操作按钮：新增、详情、修改、删除分区，所有食材绑定对应分区。

### 3\.5 食材烹饪管理页面

核心后台管理页面，管理所有用户提交的食材烹饪记录：

1. 顶部食材名称搜索框，支持模糊查询；

2. 表格字段：序号、食材名称、食材图片、存放分区、所需数量、烹饪日期、用户账号、用户姓名；

3. 批量勾选多选框，批量操作；单条操作：新增、详情、修改、删除；

4. 可查看全部用户的食材使用、烹饪记录。

## 模块 4：文件上传通用模块

# 七、核心业务流程

## 7\.1 用户前台完整操作流程

1. 用户访问前台首页，未登录自动跳转注册 / 登录页面；

2. 新用户填写账号、姓名、头像等信息完成注册；

3. 输入账号密码登录，后端生成 JWT Token 返回前端，本地缓存；

4. 首页浏览轮播图，侧边导航进入食材百科页面，搜索 / 浏览食材卡片；

5. 点击食材查看详情，可点击食材烹饪生成烹饪记录；

6. 进入个人中心，打开新增食材弹窗，上传图片、填写分区、数量、保质期保存食材；

7. 可对本人录入食材执行修改、删除操作；

8. 浏览食品资讯、健康食谱；退出登录清除 Token 返回登录页。

## 7\.2 管理员后台食材烹饪记录管理流程

1. 管理员登录后台，校验账号密码获取管理员 Token；

2. 左侧菜单栏打开【食材烹饪管理】页面；

3. 顶部输入食材名称进行模糊搜索，表格分页展示所有用户烹饪记录；

4. 支持单条查看详情、新增记录、修改记录、删除记录；

5. 表格多选框批量勾选多条数据，批量管理；

6. 同步可进入食材位置、资讯、轮播图等模块维护基础数据。

## 7\.3 JWT 登录鉴权安全流程

1. 登录接口校验账号密码成功后，后端生成携带用户 ID、用户类型（管理员 / 用户）的 JWT 令牌；

2. 前端所有业务请求，Axios 拦截器自动将 Token 放入请求头；

3. 后端全局 Jwt 拦截器拦截全部接口；

4. 无 Token、Token 过期、非法 Token 统一返回 code=401；

5. 前端响应拦截捕获 401，清空本地 Token，强制跳转登录页面。

# 八、开发与部署环境

## 8\.1 本地开发环境

### 后端环境

1. JDK 版本：1\.8

2. Maven：3\.6\+

3. MySQL：8\.0

4. IDE：IDEA

5. 启动步骤：

    - 导入 Maven 项目，修改 application\.yml 数据库连接地址、账号密码、文件上传路径；

    - 执行项目 SQL 初始化脚本，创建数据库与全部数据表；

    - 运行 DietApplication 启动类，后端服务默认端口 8080；

    - 访问接口文档地址：[http://localhost:8080/doc\.html](http://localhost:8080/doc.html)

### 前端环境

1. Node\.js：16\.x

2. IDE：VS Code

3. 启动步骤：

    - 前端目录执行`npm install`安装项目依赖；

    - 修改 axios 全局请求地址为本地后端接口地址；

    - 执行`npm run dev`启动前端开发服务；

    - 前台访问地址：[http://localhost:8081](http://localhost:8081)；后台管理地址：[http://localhost:8081/admin](http://localhost:8081/admin)

## 8\.2 Linux 服务器生产部署

1. **后端 SpringBoot 部署**

    - Maven 执行打包命令：`clean package -DskipTests`，生成 jar 包；

    - 将 jar 包上传至 Linux 服务器，后台运行命令：`java -jar diet.jar &`；

    - 防火墙开放 8080 后端端口，配置开机自启脚本。

2. **Vue 前端部署**

    - 前端执行打包：`npm run build`，生成 dist 静态文件；

    - 将 dist 文件夹上传至 Nginx 静态资源目录；

    - Nginx 配置区分前台 / 后台路由，反向代理 /api 请求转发至后端 8080 端口。

# 十、项目总结

本健康饮食管理系统基于 SpringBoot\+Vue 标准前后端分离架构开发，面向大众健康饮食场景，划分用户前台浏览端与管理员数据后台两大端口。系统完整覆盖食材百科、健康食谱、食品科普资讯、食材存放分区、个人食材录入、食材烹饪记录等核心业务功能，适配普通用户日常饮食查询、食材管理需求，同时支持管理员一站式后台数据维护。

项目代码分层清晰、模块化程度高，采用 JWT 实现安全登录鉴权，MyBatis\-Plus 简化数据库操作，封装统一请求与异常处理，内置图片上传、分页、模糊搜索等通用基础能力，易于二次开发与功能拓展。页面 UI 贴合饮食健康主题，操作流程简单直观，可直接落地部署，适用于校园、社区健康饮食科普、家庭食材管理等多种使用场景。
